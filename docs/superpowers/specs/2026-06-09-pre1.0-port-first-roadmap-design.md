# Pre-1.0 roadmap: port-first to C++ primary (umbrella) — 2026-06-09

> Umbrella spec for the full-polish path to 1.0. Each phase gets its own
> spec → plan → implementation cycle; this doc fixes the decisions, sequencing,
> gates, and success criteria so they aren't relitigated per phase.

## Decisions (locked 2026-06-09)

1. **Publish near 1.0** — no early alpha release; the repo stays private until
   the quality bar is met. Accepted risk: the Inductor-MPS / vllm-metal window
   may narrow over ~4 months.
2. **C++ MLIR path becomes primary.** The Python MSL emitter is frozen as the
   reference oracle for differential testing — bugfixes only, no new features.
3. **Perf bar**: every measured kernel family ≥85% of its roofline bound
   (memory-bound families measured against the published 546 GB/s roof;
   compute-bound against the estimated roof with the estimate caveat
   documented). MLX wall/wall ratios published alongside as honest secondary.
4. **Feature bar**: ALL implementable gaps close — `test_core` skips reduce to
   hardware-impossible only (fp64, mxfp/fp8 bias variants, 64-bit atomics,
   device printf, tf32 modes). Refusals remain only for hardware-impossible.
5. **Ratchet rule**: every commit holds upstream `test_core` (fresh cache,
   serial) at 5,335/0 minimum; the count only moves UP as skips lift. Project
   suite always 0 failed. The legacy text-parser fallback is retired (refuse).
6. Approach A — port first; MEPT lands directly in MLIR; nothing lands twice.

## Phase 0 — Hygiene (~1 wk)

Cache key gains codegen version + MEPT flag. Regenerate reports at HEAD
(README, PERFORMANCE_METRICS, upstream JSON). Pin `triton==3.7.0`. Untrack
build artifacts. Pick distribution name (PyPI `triton-metal` is squatted).
Legacy ttgir_parser → loud refusal. for_iv hang → refuse (don't hang).
Gate: clean repo, evidence ratchet in CI script, 5,335/0.

## Phase 1 — C++ port with corpus ratchet (~4–6 wk)

Port generic op-by-op lowerer to MLIR passes (extend ElementwiseOpToLLVM).
Differential harness: same kernel through C++ + Python, compare buffers.
Op families in coverage order: elementwise/cast → reduce/scan → load/store/
atomics → softmax/layernorm/dot templates. C++ default-on at parity.
Gate: 5,335/0 via C++; differential pass on full corpus; Python frozen.

## Phase 2 — MEPT spine in MLIR (~3–4 wk)

Register-array per-thread model + convert_layout shuffle natively in MLIR;
>1024 ceiling lifted; chained_reductions, sizePerThread>1 paths.
Gate: large-shape skips become passes; ratchet only up.

## Phase 3 — Feature completeness in C++ (~3–4 wk)

FA HEAD_DIM=64; i64 reduce/where/transpose; fp16 atomics via word-CAS;
trans_4d; cat/join N-D; cf-dialect; map_elementwise. Refusals → hw-only.
Gate: test_core skips hw-impossible only; FA tutorial parity.

## Phase 4 — Roofline grind (~3 wk)

Per family: reduce_sum (16%→85%+), vector_add (72%→85%+), FA roofline
+ MLX bench, launch overhead halved, scan/sort/CE benchmarked. Methodology:
symmetric wall/wall + GPU-time roofline.

## Phase 5 — 1.0 gates

All gates green; 7-day claim-vs-evidence freeze; rename + publish.

## Risks

C++ ratchet stall (mitigate: Phase 1 is family-incremental); roofline 85%
unreachable somewhere (escalate with evidence; bar set per-family); window
loss (accepted); solo bus-factor (specs/diff-tests are durable).
